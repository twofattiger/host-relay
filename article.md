# 零成本、免公网IP！基于 Cloudflare 打造极简 Web SSH 管理面板

> 💡 **提示：** 文章末尾附有项目源码，如需获取本方案更详细的代码内容，关注公众号并发送 "主机中继"。

对于手里有几台 NAS、软路由或者云服务器的玩家来说，想要随时随地连上 SSH 进行管理，通常只有两种选择：
要么花钱买一台带公网 IP 的 VPS 自己搭内网穿透（比如 FRP），但维护成本和服务器费用是一笔持续的开销；要么使用第三方提供的现成管理工具，但把核心机器的底层权限交到别人手里，数据和隐私的安全始终是个隐患。

那有没有一种两全其美的方案：**既能不花一分钱买服务器，又能拥有一个数据完全自主可控、能在网页上直接 SSH 管理所有机器的轻量面板？**

今天给大家分享一个非常实用的方案：基于 Cloudflare Workers 和 Durable Objects (DO) 构建的轻量级主机管理面板 `host-relay`。它就像一根隐形的网线，不仅能实时监控主机的 CPU、内存状态，还能直接在浏览器里打开终端进行管理。

---

## 🌟 方案优势？

### 1. 纯反向隧道，无视 NAT
传统的 SSH 需要机器暴露在公网上。而 `host-relay` 采用了类似 FRP 的**反向中继模型**。
客户端（Agent）常驻后台，主动向外建立长连接。这就意味着不管你的机器是在几层路由器的 NAT 后面，只要它能上网，就能穿透出来。你只需要打开浏览器，就能直接敲命令。

### 2. 极简部署与真正的“零成本”
整个服务端被压缩在了一个单文件 `worker.js` 中。利用 Cloudflare Durable Objects 保持 WebSocket 长连接和内置的 SQLite 存储状态。
**最关键的是：** 它完全卡在 Cloudflare 的免费额度内设计。状态推送、SSH 字节流中继等开销极低，闲置时连接自动挂起进入 hibernation 状态不计费，个人 Homelab 规模用起来可以说是白嫖无忧。

### 3. 硬核的私钥级安全
把内网机器暴露出来，安全永远是第一位的。除了常规的密码认证（密码经 AES-GCM 加密存储），这个方案还支持最高级别的**私钥免密认证**。
- **本地静默验证**：你在机器本地指定好私钥文件，网页端连接时选择“私钥”模式，Agent 会直接读取本地文件进行登录。
- **隐私拉满**：私钥内容和 SSH 口令绝不经过网络，也不会上传给 Cloudflare 服务端，从根本上杜绝了泄露可能。
- **防滥用**：SSH 目标地址在客户端被强制白名单化，有效防止 SSRF 攻击。

---

## ⚙️ 本应用的核心原理

整个交互流程其实非常干脆：

1. **状态推送**：浏览器通过 WSS 连接到 Hub DO。各个主机的 Agent 定期上报状态给 Host DO，再汇总给 Hub DO 进行广播。这就实现了状态的实时推送，而不是低效的轮询。
2. **反向 SSH 隧道**：点击“管理”时，Worker 签发一次性短时 Ticket（带 Nonce 防重放）。浏览器携带 Ticket 连入对应的 Host DO。
3. **终端复用**：Host DO 扮演桥梁角色，将网页端 `xterm.js` 产生的字节流，通过之前 Agent 建立的反向 WebSocket 隧道转发回去。
4. **本地终结**：Agent 收到指令后，直接在本地利用 Go 的 `x/crypto/ssh` 库拨号本机 22 端口，打开 PTY 进行交互。

## 🚀 三步搞定部署（3分钟搞定）

部署这套工具同样是薅 Cloudflare 的羊毛。跟着下面的步骤，最多三分钟就能把这个专属面板跑起来。

> **注意：** 由于用到了 Durable Objects 的内置 SQLite，首次部署必须通过命令行工具来划拨物理节点和执行数据库迁移。

### 步骤 1：准备与登录
准备好本地 Node.js 环境，克隆好代码，进入 `src/worker` 目录。
打开终端，运行登录命令，浏览器会弹出来让你授权：
```bash
npx wrangler login
```

### 步骤 2：部署
一行命令直接起飞：
```bash
npx wrangler deploy
```

### 步骤 3：上把锁（配置 Secret）
作为一个私密的管理面板，肯定不能让别人随便看。部署完成后，必须通过命令（或在网页控制台的 Settings -> Variables & Secrets 中）注入 4 个核心密钥：

```bash
# 面板登录密码
npx wrangler secret put ADMIN_PASSWORD

# 会话 Cookie 签名密钥（随便敲一长串字符）
npx wrangler secret put SESSION_SECRET

# SSH Ticket 签名密钥（随便敲一长串字符）
npx wrangler secret put TICKET_KEY

# 保存密码用的加密密钥（随便敲一长串字符）
npx wrangler secret put ENC_KEY
```

服务端搞定后，登录面板点击右上角的**“添加主机”**，会给你一行包含专属 Token 的启动命令。
在目标主机上下载编译好的 Agent 并执行该命令，你的主机就会在面板上瞬间上线了！

## 💡 强烈建议：绑定自定义域名

如果你在国内访问，大概率会发现刚才生成的 `*.workers.dev` 链接经常受网络干扰打不开。
解决办法很简单：进入 CF 控制台，找到你刚部署的 Worker，在 **Settings -> Domains & Routes** 里，绑定一个你自己托管在 CF 上的域名（Custom Domain）。几十秒解析生效后，就能配合 HTTPS 稳定、丝滑地访问你的管理面板了。

---

## 🔒 进阶玩法：私钥认证模式

如果你需要最高的安全性，不想在网页端输密码，可以在目标机器启动 Agent 时加上 `--ssh-key` 参数指向你的私钥：
```bash
./agent --server wss://你的域名 --id h_xxxx --token tk_xxxx --ssh-target 127.0.0.1:22 --ssh-key /root/.ssh/id_rsa
```
这样网页端直连时，选择“私钥”验证，一切就全在本地静默完成了，不用担心任何凭据泄露的风险。
